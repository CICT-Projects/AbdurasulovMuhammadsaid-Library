## 📖 Library - Система Управления Библиотекой

Полнофункциональное веб-приложение для управления книгами и авторами.

### Архитектура

**Backend:**
- `.NET 10 Web API`
- `Entity Framework Core` с `SQLite`
- `CORS` включен для локального desarrollo
- REST API на `http://localhost:5000`

**Frontend:**
- `React 19` с `Vite`
- `React Router` для навигации
- Современный дизайн с CSS стилями

### Структура проекта

```
backend/
├── Models/                    # Модели данных
│   ├── Book.cs
│   └── Author.cs
├── Controllers/               # API контроллеры
│   ├── BooksController.cs
│   └── AuthorsController.cs
├── Data/
│   └── AppDbContext.cs       # EF Core DbContext
├── Program.cs                # Точка входа и конфигурация
└── library.db               # SQLite база данных (создается автоматически)

frontend/
├── src/
│   ├── Books.jsx            # Компонент для управления книгами
│   ├── Authors.jsx          # Компонент для управления авторами
│   ├── App.jsx              # Главный компонент с Router
│   ├── App.css              # Стили приложения
│   └── index.css            # Глобальные стили
└── dist/                    # Production build
```

### 🚀 Установка и запуск

#### Backend

1. **Перейдите в папку backend:**
```bash
cd backend
```

2. **Восстановите зависимости и запустите:**
```bash
dotnet run
```

Backend запустится на `http://localhost:5000`

База данных `library.db` будет создана автоматически при первом запуске.

#### Frontend

1. **В другом терминале, перейдите в папку frontend:**
```bash
cd frontend
```

2. **Установите зависимости (если еще не установлены):**
```bash
npm install
```

3. **Запустите dev сервер:**
```bash
npm run dev
```

Frontend будет доступен на `http://localhost:5173`

### 📋 API Endpoints

#### Books (Книги)
- `GET /api/books` - получить все книги
- `GET /api/books/{id}` - получить книгу по ID
- `POST /api/books` - создать новую книгу
  ```json
  {
    "title": "Название книги",
    "year": 2024
  }
  ```
- `DELETE /api/books/{id}` - удалить книгу

#### Authors (Авторы)
- `GET /api/authors` - получить всех авторов
- `GET /api/authors/{id}` - получить автора по ID
- `POST /api/authors` - создать нового автора
  ```json
  {
    "name": "Имя Автора",
    "country": "Страна"
  }
  ```
- `DELETE /api/authors/{id}` - удалить автора

### 🎨 Функциональность Frontend

**Страница /books**
- Просмотр списка всех книг
- Форма для добавления новой книги (название, год)
- Кнопка удаления для каждой книги
- Автоматическое обновление списка после действий

**Страница /authors**
- Просмотр списка всех авторов
- Форма для добавления нового автора (имя, страна)
- Кнопка удаления для каждого автора
- Автоматическое обновление списка после действий

### ⚙️ Технические детали

**Backend:**
- Использует Microsoft.EntityFrameworkCore.SQLite для работы с БД
- Автоматическое создание схемы БД при старте (EnsureCreated)
- CORS настроен для принятия запросов с любого источника
- Возвращает `NotFound (404)` для несуществующих ресурсов

**Frontend:**
- Используется Fetch API для HTTP запросов
- React Router для клиентской маршрутизации
- Обработка ошибок и состояния загрузки
- Адаптивный дизайн (мобильные и десктопные устройства)

### 🔧 Разработка

### 📝 Notes

- База данных хранится в файле `backend/library.db` (SQLite)
- При перезапуске Backend данные сохраняются в БД
- Frontend работает с Backend через CORS
- Оба приложения настроены для локальной разработки
