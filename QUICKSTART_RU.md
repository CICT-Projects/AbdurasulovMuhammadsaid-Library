# 🚀 QUICK START - Быстрый старт

## ⚡ За 2 минуты до запуска

### Windows
Просто двойной клик на файл:
```
start.bat
```

Готово! Откройте в браузере: **http://localhost:5173**

### Linux/Mac
```bash
chmod +x start.sh
./start.sh
```

Откройте в браузере: **http://localhost:5173**

## 📖 Как пользоваться

### 📚 Управление Книгами (страница /books)

1. **Добавить книгу:**
   - Введите название (например: "1984")
   - Введите год (например: 1949)
   - Нажмите "Добавить книгу"

2. **Посмотреть список:**
   - Все книги отображаются ниже формы

3. **Удалить книгу:**
   - Нажмите кнопку "Удалить" рядом с книгой

### ✍️ Управление Авторами (страница /authors)

1. **Добавить автора:**
   - Введите имя (например: "Джордж Оруэлл")
   - Введите страну (например: "Великобритания")
   - Нажмите "Добавить автора"

2. **Посмотреть список:**
   - Все авторы отображаются ниже формы

3. **Удалить автора:**
   - Нажмите кнопку "Удалить" рядом с автором

### 🧭 Навигация

- Нажмите "📖 Library" - вернется на главную
- Нажмите "Книги" - откроется страница книг (/books)
- Нажмите "Авторы" - откроется страница авторов (/authors)

## 🧪 Проверка работы

Всё работает если:

✅ Форма принимает данные
✅ Список обновляется после добавления
✅ Кнопка удаления удаляет запись
✅ При обновлении страницы данные остаются (сохранены в БД)
✅ Нет ошибок в консоли браузера (F12)

## ⚠️ Проблемы?

### Приложение не запускается

1. **Убедитесь что установлено:**
   - .NET 10 (для Backend)
   - Node.js (для Frontend)

2. **Попробуйте:**
   ```bash
   cd backend && dotnet build
   cd ../frontend && npm install
   ```

3. **Потом запустите start.bat/start.sh снова**

### API недоступен

- Проверьте, что Backend запущен (первое окно терминала)
- Подождите 5-10 секунд после запуска
- Проверьте что порт 5000 свободен

### Frontend не загружается

- Проверьте URL: http://localhost:5173
- Подождите 5-10 секунд (компиляция может быть медленной)
- Откройте F12 и проверьте консоль на ошибки

## 💾 Данные сохраняются в

Файл: `backend/library.db` (SQLite база данных)

При каждом запуске приложения новые данные добавляются в этот файл.

Если хотите начать с нуля - удалите файл `library.db` и запустите приложение снова.
